# UI Test Cases

## Применённые техники тест-дизайна

### Equivalence Partitioning (EP)

Использована для проверки функциональности авторизации.

Классы эквивалентности:

- валидные учетные данные;
- невалидный логин;
- невалидный пароль;
- пустые значения.

Form Authentication
TC-LOGIN-001. Успешная авторизация с валидными данными

Предусловия:

Открыта страница Form Authentication.

Шаги:

Ввести логин tomsmith.
Ввести пароль SuperSecretPassword!.
Нажать кнопку Login.

Ожидаемый результат:

Пользователь успешно авторизован.
Отображается сообщение об успешном входе.
Выполнен переход на защищённую страницу.

Фактический результат:

Соответствует ожидаемому результату.

Статус: Pass

TC-UPLOAD-002. Попытка загрузки без выбора файла

Предусловия:

Открыта страница File Upload.

Шаги:

Не выбирать файл.
Нажать кнопку Upload.

Ожидаемый результат

Система отображает понятное сообщение пользователю:

«Файл не выбран», либо «Необходимо выбрать файл для загрузки».

Фактический результат

Отображается страница: Internal Server Error

Вкладка Network показывает ответ:

POST /upload
500 Internal Server Error

Статус: Fail
