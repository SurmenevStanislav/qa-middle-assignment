# MongoDB

## 1. Как бы ты хранил данные о бронированиях в MongoDB?

Я бы использовал отдельную коллекцию `bookings` и хранил основные данные бронирования в одном документе.

Пример структуры:

```json
{
  "_id": ObjectId("..."),
  "user": {
    "id": 1,
    "name": "John Smith",
    "email": "john@example.com"
  },
  "room": {
    "id": 10,
    "type": "Deluxe"
  },
  "checkin": "2025-06-01",
  "checkout": "2025-06-05",
  "status": "confirmed",
  "created_at": "2025-05-20T10:00:00Z"
}
```

Для часто используемых данных (имя пользователя, тип комнаты) я бы применил embedded-подход. Это уменьшает количество запросов и упрощает чтение документа.

Если информация о пользователях и комнатах изменяется независимо и должна храниться централизованно, можно использовать referenced-подход через идентификаторы пользователей и комнат.

Для системы бронирования я бы выбрал комбинированный вариант: хранить ссылки на основные сущности и дублировать наиболее востребованные данные для быстрого чтения.

## 2. Какие индексы создал бы?

Я бы создал следующие индексы:

- `user.id` — для быстрого поиска бронирований пользователя.
- `room.id` — для поиска бронирований конкретной комнаты.
- `status` — для выборки подтверждённых или отменённых бронирований.
- `checkin` и `checkout` — для поиска по диапазонам дат.
- Составной индекс `(room.id, checkin, checkout)` для проверки пересечения бронирований одной комнаты.

Такие индексы ускоряют основные бизнес-сценарии системы бронирования.

## 3. Как проверил бы корректность данных?

Через MongoDB Compass или mongo shell я бы проверял:

- наличие обязательных полей;
- корректность типов данных;
- отсутствие пустых значений в критичных полях;
- корректность статусов бронирования;
- корректность дат checkin и checkout;
- отсутствие логически некорректных данных, например checkout раньше checkin;
- соответствие данных между связанными коллекциями.

Для выборочной проверки использовал бы запросы по пользователю, комнате, статусу и диапазону дат, а также сравнивал результаты с данными, отображаемыми в приложении.
